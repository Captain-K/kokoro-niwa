# CLAUDE.md — 3D枯山水エディタ「心庭（こころにわ）」

## プロジェクト概要

AIハッカソン2026予選2ndラウンド Bブロック提出作品。制限時間3時間で完成させる**3D枯山水ガーデンエディタ**。ユーザーが3D空間で岩・砂紋・苔を配置して自分だけの枯山水を作り、AIが庭の意味を解説。最後に**審査員（福岡俊弘氏）の3Dアバター**を庭に召喚して記念撮影できる。

**テーマ**：「ZEN」
**チーム**：黒木・大貫
**締切**：2026年4月25日 16:15

## 絶対に守るルール

1. **動くものを最優先**。完璧より完成。60分でMVP、90分でブラッシュアップ、15分でテスト。
2. **既存の技術スタックから外れない**（Three.js + Vanilla JS + Gemini API）。フレームワーク追加禁止。
3. **シングルHTMLファイルで完結させる**。GitHub Pagesで即公開するため。
4. **画像生成API呼び出しは最小限**。当日止まるリスクあり（過去の教訓）。テクスチャは事前に用意済みを使う。
5. **AI呼び出しは失敗時のフォールバック必須**。AIが落ちても遊べる状態を維持する。
6. **git commitは動いた瞬間ごと**。「MVP完成」「岩配置OK」「AI解説OK」等のタグを打つ。
7. **エラーはそのまま貼って相談**。自分で解釈しない。

## 技術スタック

- **3D**: Three.js r128（CDN読み込み、`https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js`）
- **AI**: Gemini API（`gemini-2.5-flash`または最新の高速モデル）
- **ホスティング**: GitHub Pages（黒木担当）
- **言語**: Vanilla JS（TypeScript不要、ビルドプロセスなし）
- **UI**: HTML + CSS（フレームワーク不要）
- **音**: Web Audio API + 事前録音した効果音（鹿威し、砂の音、鐘の音）

## 禁止事項

- `THREE.CapsuleGeometry`の使用（Three.js r128に存在しない）
- `localStorage`の使用（ハッカソン環境で動作保証なし）
- npm/yarnのインストール（ビルドなしで動かす）
- React/Vue/その他フレームワークの追加
- 複雑なゲームロジックの実装（3時間で完成しない）
- リアルタイム画像生成への依存（当日止まる）

## コーディング規約

- 関数は1つ20行以内を目安
- 3D座標系は **Y-up**（Three.jsデフォルト）
- 庭の中心を原点(0,0,0)、サイズは**8x8ユニット**の正方形
- カメラ初期位置: `(5, 5, 5)`、look at `(0, 0, 0)`
- コメントは日本語OK
- console.logはデバッグ後に削除

MVP達成後、ブラッシュアップフェーズへ。

## ブラッシュアップ優先順位（90分）

1. **砂紋の描画**（熊手ツールで円周状の模様）
2. **効果音の追加**（配置時、砂紋時、AI解説完了時）
3. **AI解説の質向上**（庭全体を見てプロンプト生成）
4. **福岡さんアバター召喚ボタン**（庭完成後、アバターが歩いて登場）
5. **記念撮影モード**（カメラ固定＋スクリーンショットDL）
6. **庭の命名機能**（AIが庭に名前を付ける）
7. **苔オブジェクトの追加**（オプション）

## デモで見せる流れ（3分プレゼンのデモ90秒分）

1. 何もない3D砂地 → 「あなたも庭を作れます」
2. 審査員に岩を配置してもらう（マウスクリック）
3. AIが即座に「この配置は○○を表しています」と解説
4. 砂紋を描いて完成
5. **「福岡さんを召喚！」ボタン** → 3Dアバターが歩いて登場
6. アバターが庭の中心で一礼
7. **記念撮影ボタン** → スクショが保存される
8. 「**この写真、Twitterに上げてください！**」で締める

## よくあるバグと対策

- **Three.jsのCORS**: テクスチャ読み込みは同一オリジンから。GitHub Pagesに置く。
- **Gemini API CORS**: 直接ブラウザから呼ぶとCORSで弾かれる可能性 → **Gemini API KeyをURL paramで渡す公式エンドポイント**（`generativelanguage.googleapis.com`）を使う
- **モバイル対応**: 今回は不要。デスクトップChromeのみ想定。
- **カメラ操作**: OrbitControlsが効かない場合は、Three.jsのr128版URLと合ってるか確認

