# PROMPTS.md — Claude Codeに渡す段階別プロンプト集

3時間で完成させるため、**段階的にClaude Codeに指示を出す**ためのプロンプト集。
各プロンプトは順番に投げる。前のフェーズが完成してから次へ。

```
このプロジェクトは3時間のAIハッカソン用です。CLAUDE.md と SPEC.md をまず読んでください。
読み終わったら「準備完了」とだけ返してください。
```

## 🏗️ Phase 1: MVP構築（0〜60分）

### Prompt 1-A: 骨格作成（5分）

```
index.html を1ファイルで作ってください。以下を含めてください：

1. Three.js r128 を CDN から読み込む
2. OrbitControls を examples から読み込む（r128対応版）
3. body いっぱいに #canvas を表示
4. 下部に toolbar（岩・砂紋・苔・AI解説・撮影のボタン）
5. 下部に AI発言用の吹き出しエリア
6. ヘッダーに「心庭 —こころにわ—」のタイトル
7. Noto Serif JP フォントを Google Fonts から読み込む
8. 背景色 #f5f0e8、和紙っぽい雰囲気

CSSとJSは全部index.htmlに入れてOK。外部ファイルは作らない。
完成したらプレビューURLを教えてください。
```

### Prompt 1-B: 3Dシーン基本（15分）

```
次に3Dシーンを作ります：

1. THREE.Scene を作成、背景色 #f5f0e8
2. PerspectiveCamera(50, aspect, 0.1, 100)、初期位置 (5, 5, 5)、lookAt(0,0,0)
3. WebGLRenderer、antialias:true、影有効
4. DirectionalLight (強度1.0, 位置(5,10,5)) と AmbientLight (強度0.4)
5. 8x8 の PlaneGeometry で砂地（128x128セグメント）、色 #ede4cf、影を受ける
6. 砂地の四方に木枠（BoxGeometry、色 #3e2723）
7. OrbitControls、minDistance 3、maxDistance 15、maxPolarAngle 1.5
8. アニメーションループで renderer.render

完成したら砂地が3Dで表示されてマウスで回せる状態にしてください。
```

### Prompt 1-C: 岩配置機能（15分）

```
岩配置機能を実装します：

1. ツールバーの「🪨岩」ボタンでモード切替
2. 岩モードON中に砂地クリックで岩配置
3. 岩は IcosahedronGeometry(random(0.3, 0.6), 0)、MeshStandardMaterial、色は #6b6760〜#8a7f72ランダム、flatShading:true
4. 配置時にY軸ランダム回転、スケール 0.7〜1.3 ランダム
5. 影あり（castShadow, receiveShadow）
6. Raycaster で砂地との交点を正確に取る
7. 配置した岩を rocks 配列で管理

動作確認: クリックで違う形の岩が並ぶ。
```

### Prompt 1-D: Gemini API統合（15分）

```
AI解説機能を実装します：

1. 「🧙AI解説」ボタンで現在の庭をAIに解説させる
2. Gemini API エンドポイント: https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent
3. APIキーは const API_KEY = '...' で最上部に定義（後で差し替える）
4. プロンプトは SPEC.md の「庭の解説プロンプト」を参照
5. 結果をAI吹き出しエリアにタイプライター風に1文字ずつ表示（50ms間隔）
6. API失敗時は以下のフォールバック配列からランダム：
   - "静寂の中に、確かな存在感がございますな。"
   - "この配置、何か語りたげに見えます。"
   - "庭は心を映す鏡。見事でございます。"
   - "石の間に、目に見えぬ線が引かれておりますな。"
   - "余白こそ、庭の真髄でございます。"

API_KEY未設定でもフォールバックで動くこと。
```

## 🎨 Phase 2: ブラッシュアップ（75〜180分）

### Prompt 2-A: 砂紋描画（30分）

```
砂紋描画機能を実装します：

1. ツールバーの「🌀砂紋」ボタンでモード切替
2. 砂紋モード中、マウスドラッグで砂地の頂点Y座標を凹ませる
3. ドラッグ中は砂地との交点周辺の頂点を、距離に応じて最大-0.08下げる
4. 距離の影響範囲は0.3ユニット、ガウス分布で減衰
5. 頂点変更後は geometry.attributes.position.needsUpdate = true と computeVertexNormals() 呼ぶ
6. 岩配置時に自動で同心円の砂紋を描く（岩の周り0.5〜1.2ユニットにsin波状の凹凸）

砂紋描画中はマウスカーソルを「crosshair」に変更。
```

### Prompt 2-B: 効果音（20分）

```
効果音を実装します：

sounds/ フォルダに以下を配置する想定（なければfreesoundからダウンロードの指示）：
- rock_place.mp3 (岩配置時)
- sand_rake.mp3 (砂紋描画中、ループ)
- bell.mp3 (AI解説完了時)
- camera.mp3 (撮影時)
- shishi_odoshi.mp3 (ランダムBGM)

実装：
1. const sounds = { rock: new Audio('sounds/rock_place.mp3'), ... } でプリロード
2. 岩配置時 sounds.rock.play()
3. 砂紋描画中は sand_rake をloopで、描画停止で停止
4. AI解説完了時に bell 再生
5. 30秒ごとにランダムで shishi_odoshi 再生

音が見つからない場合はフリー素材サイトから取ってくるか、
Web Audio API で合成音を作ってもOK。
```

### Prompt 2-C: 福岡さんアバター召喚【目玉機能】（40分）

```
福岡さんアバター召喚機能を実装します。これが本アプリの目玉です。

アバターモデル：
- assets/fukuoka_avatar.glb があればそれを使う
- なければシンプルな人型を作る：
  - 頭: SphereGeometry(0.15), 色 #f5c89a
  - 胴体: CylinderGeometry(0.2, 0.25, 0.7), 色 #4a3c2a（僧衣イメージ）
  - 腕: CylinderGeometry(0.06, 0.06, 0.5) × 2
  - 脚: CylinderGeometry(0.08, 0.08, 0.6) × 2
  - 頭の上に「福岡」テキストスプライト（CanvasTextureで作成）
  - 全体をGroupでまとめる

召喚演出：
1. 「👤福岡さん召喚」ボタンを押す
2. アバターを (4, 0, 4) に配置、初期 visible=false
3. パーティクル効果（SphereGeometry × 20個を上に飛ばす）で「煙」
4. 1秒後、アバター visible=true
5. 2秒かけて (0, 0, 2) に歩く（position.lerp）
6. 到着後、一礼アニメ：rotation.x を 0→-0.5→0（1秒）
7. アバター頭上に吹き出し：「ほう、見事な庭でございますな」
8. AI吹き出しエリアにも同じセリフ

アバターが庭を歩く時、少し上下に揺れる（歩行モーション）と良い。
```

### Prompt 2-D: 記念撮影機能【目玉機能】（30分）

```
記念撮影機能を実装します。

1. 「📸撮影」ボタンを押すとカウントダウン表示「3...2...1...」（画面中央、大きく）
2. 「カシャッ」音再生
3. 画面全体を一瞬白くフラッシュ（白いdivをopacity 0→1→0、計300ms）
4. renderer.domElement.toBlob() でキャンバス画像取得
5. 新しいCanvasで画像に以下を合成：
   - 右下に「心庭 — こころにわ —」の文字（Noto Serif JP、白色、影付き）
   - 日付「2026.04.25」
   - 庭の名前（AIが命名済みなら表示）
6. toBlob → URL → download="心庭-YYYYMMDD-HHMMSS.png" で自動DL
7. DL後、AI吹き出しに「よき一枚が撮れましたな」

撮影時は toolbar と header を一時非表示にする（display:none）。
撮影完了後に表示戻す。
```

### Prompt 2-E: 庭の命名機能（15分）

```
AI解説の後に「✨この庭に名前を」ボタンを追加します：

1. ボタン押下で Gemini に以下のプロンプトを送る：
   "以下の枯山水に詩的な名前を付けてください。「○○庭」の形式で、8文字以内の漢字のみ。
   他の文字は一切含めず、名前だけ返してください。
   岩の数: N個、砂紋: M本"

2. 返ってきた名前を gardenName 変数に保存
3. ヘッダー下に「〜 ${gardenName} 〜」と大きく表示
4. 撮影時にこの名前も画像に合成

フォールバック：
- API失敗時は以下からランダム：["静寂五岳庭", "月影苔庭", "無心波紋庭", "一期一会庭", "無辺無際庭"]
```

### Prompt 2-F: 仕上げ（15分）

```

6. コンソールにエラーが出ていないか最終チェック

完成したら共有URLを教えてください。
```

## 💡 困ったときのプロンプト

### AI連携がうまくいかないとき

```
Gemini APIの呼び出しが失敗しています。以下のエラーメッセージです：

[ここにエラーをコピペ]

CORSエラーか、APIキー問題か、エンドポイントの問題か、調査して修正してください。
フォールバックは動作しているので、APIなしでもデモは可能な状態は維持してください。
```

### 3Dがうまく動かないとき

```
Three.jsで以下の問題が起きています：

[現象を具体的に]

ブラウザのコンソールエラー:
[コンソールの内容をコピペ]

Three.js r128の制約内で解決してください。CapsuleGeometryなどr142以降の機能は使えません。
```

### パフォーマンスが重いとき

```
画面がカクついています。以下を試してください：

1. 岩のgeometryを共有する（同じgeometryを使い回す）
2. 影の解像度を下げる（shadowMap.mapSize = 1024）
3. 砂紋の頂点更新を間引く（3フレームに1回）
4. パーティクル数を減らす

軽量化の提案をコードで示してください。
```

## 🚨 緊急時のフォールバック

万が一3時間で完成しない場合の段階的縮小：

### レベル1: 福岡さんアバター間に合わない
→ 汎用人型プリミティブ + 名札「審査員」で進める

### レベル2: 砂紋実装が複雑すぎる
→ TubeGeometryで線を貼るだけに変更

### レベル3: AI API設定でつまづく
→ フォールバック文のみで進める、「AI」と名乗るだけ

### レベル4: 3D全体がダメ
→ 2Dキャンバスに切り替え、「3D風」UIで妥協

**最低限これだけあればプレゼンできる最小形**：
- 3D砂地
- 岩を配置できる
- AIっぽい解説文が出る（フォールバックでもOK）
- アバター的なものが出現する
- スクショが撮れる（`canvas.toDataURL`でも可）
